# ClarityCoach AI — План завершення проєкту

**Мета**
Завершити MVP мобільного додатку, який веде користувача від формування профілю і великої мети до цілей GROW+SMART, чек-інів, нагадувань і базової аналітики прогресу. Усі дані зберігаються локально (offline-first).

**MVP Scope (обовʼязково)**
- Onboarding → Dashboard → Profile → North Star → Goals (GROW+SMART) → Check-ins → Analytics.
- Локальне збереження всіх даних і відновлення після перезапуску.
- Нагадування з локальними нотифікаціями.
- Базова аналітика прогресу і історія.
- AI-підказки через Google Gemini API (мінімальний набір сценаріїв).
- UA/EN локалізація базових екранів.

**Повний перелік вимог з ТЗ (включно з розширеннями після MVP)**
- Партнерська підзвітність: запрошення наставника/лідера, режим тільки перегляд прогресу, сповіщення партнеру.
- Інтеграція з календарем: синк дедлайнів, чек-інів і фокус-блоків.
- Журнал дій/лог: фіксація виконаних дій і статусів по кроках.
- Ресурси/перешкоди: окремий блок у цілях + аналіз повторюваних перешкод.
- Історія минулих цілей: архів і перегляд завершених/заморожених.
- Уроки/висновки: збереження текстових ретроспектив і підсумків.
- Мультирольовість: панель ролей і швидке перемикання контексту.
- Бібліотека шаблонів: приклади ролей/цілей/шаблонів SMART.
- Регулярні звіти: щотижневі/щомісячні опитування з фіксованими питаннями.
- Експорт і шаринг: PDF/звіт або поділитися підсумком.

**Definition of Done (приймання)**
- Користувач проходить повний флоу без втрати даних.
- Можна створити, редагувати, архівувати профіль, North Star, цілі, чек-іни.
- Dashboard показує “сьогоднішній фокус” на основі активних цілей і дій.
- Check-ins мають історію і статус пропусків.
- Analytics показує прогрес у %, тренд і ретроспективу за період.
- Локальні нотифікації працюють за графіком.
- AI-сценарії працюють через Gemini API з обробкою помилок і fallback-режимом.

**Рекомендований стек (як стандарт для реалізації)**
- State management: `flutter_riverpod`.
- Routing: `go_router`.
- Локальне сховище: `drift` (SQLite) + міграції.
- Нотифікації: `flutter_local_notifications`.
- Графіки: `fl_chart` або прості кастомні віджети.
- AI: Google Gemini API + контрактний шар (fallback на локальні підказки при offline/помилках).

**Архітектура додатку**
- `lib/domain`: моделі, правила, бізнес-логіка.
- `lib/data`: репозиторії, DAO, мапери, сховище.
- `lib/features`: екрани і state.
- `lib/shared`: UI-компоненти, теми, утиліти.

**Дані (основні моделі)**
- Profile: role, scope, vision, responsibility, mission, kpis, stakeholders, values.
- RoleContext: name, description, roleProfileId, isActive.
- NorthStar: title, description, horizon, keyword, emotion, image(optional).
- Goal: title, priority, status, GROW fields, SMART fields, deadline, resources, obstacles.
- ActionItem: title, dueDate, status, blockers.
- CheckIn: cadence, date, progressPercent, blockers, lessons.
- Reminder: schedule, channel, payload.
- PartnerAccountability: partnerEmail, accessLevel, reportsSchedule.
- ActivityLog: date, actionType, relatedEntityId, status.
- Template: type, title, content, tags.
- ExportReport: period, format, filePath.

**Екрани (мінімальна функціональність)**
- Onboarding: короткий вступ, перехід у профіль.
- Profile: форма + перегляд збережених даних.
- North Star: створення/редагування, візуальний блок.
- Goals: майстерка GROW+SMART, список цілей, деталі, статуси.
- Check-ins: щоденний/тижневий, історія.
- Analytics: прогрес, тренд, ретро, історія.
- Dashboard: фокус дня, активні цілі, наступна дія.

**Нагадування та підзвітність**
- Локальні нотифікації за розкладом.
- Правило пропуску: якщо 2+ пропуски чек-інів, показувати alert.
- Лог активності: дата, дія, статус.
- Партнерська підзвітність: запрошення, дозволи, звітність.

**AI-шар (MVP заглушка або контракт)**
- Запити: уточнення ролі, переформулювання цілей, коротке резюме чек-інів.
- Підключення: Google Gemini API з окремим сервісом і контрактом.
- Fallback: локальні “підказки” або шаблони, якщо немає мережі/лімітів.

**План реалізації (етапи)**
1. Підготовка: підтвердити стек, створити структуру папок, базові залежності.
2. Дані: моделі, SQLite-схеми, репозиторії, базові CRUD.
3. Profile + North Star: повний цикл створення/редагування/перегляду.
4. Goals: GROW+SMART wizard, список, деталі, статуси.
5. Check-ins: форми, історія, базові правила пропусків.
6. Dashboard: агрегований фокус дня.
7. Analytics: прогрес у %, тренди, історія.
8. AI (Gemini API): сервіс, ключі, базові сценарії, fallback.
9. Нотифікації: локальний планувальник.
10. Локалізація: UA/EN, фінальна поліровка.
11. Тести і стабілізація.
12. Партнерська підзвітність: запрошення, доступ, звіти.
13. Інтеграція з календарем: синк дедлайнів і чек-інів.
14. Журнал дій і архів: історія дій, архів цілей.
15. Мультирольовість: панель ролей і перемикання контексту.
16. Бібліотека шаблонів: ролі/цілі/SMART пресети.
17. Регулярні звіти: щотижневі/щомісячні форми.
18. Експорт/шаринг: PDF або підсумок за період.

**Ризики**
- Відсутність чітких рішень по стеку затримує реалізацію.
- Нотифікації можуть мати нюанси на iOS/Android.
- AI-інтеграція залежить від квот/лімітів і стабільності мережі.
- Інтеграція календаря має різні обмеження на платформах.
